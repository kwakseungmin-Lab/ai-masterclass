## 1. 파이프라인/프로세스 분석

### 1.1 파이프라인

```
"길건너 친구들 만들어줘"
    ↓
[Research] 나무위키/웹에서 게임 메카닉, 장애물 종류, 아트 스타일 조사 (~3분)
    ↓
[Code] Claude Code가 Three.js + TypeScript로 게임 코드 생성 (~8분)
        에셋은 Three.js 기본 도형(Box/Cylinder)으로 코드 안에서 직접 생성
    ↓
[Assemble] vite-plugin-singlefile로 단일 HTML 패키징 (~1분)
    ↓
519KB index.html — 브라우저에서 바로 플레이
```

메탈슬러그와 달리 에셋을 외부에서 가져올 필요 없이, **리서치 → 코드 생성 → 패키징**의 3단계로 완성.
**총 소요: 아이디어 → 플레이 가능한 게임까지 약 15분**

### 1.2 수동 반복 수정 루프

한 번에 완벽한 게임이 나오진 않았음. 실제 과정:
1. 첫 빌드: 기본 기능만 동작
2. 원작 레퍼런스 스크린샷과 비교
3. 차이점 수정 (카메라, 색감, 화면비 등)
4. 세부 버그 수정 (캐릭터 방향, 나무 충돌)

**빌드 → 확인 → 수정 루프를 6회 반복**해서 원작에 가까워짐.

## 2. 기술적 발견

### 2.1 Three.js 기본 도형만으로 간단한 로우폴리 3D 게임은 만들 수 있다

Three.js의 기본 도형(BoxGeometry, CylinderGeometry 등)을 조합하면 외부 3D 모델 파일 없이도 게임 에셋을 만들 수 있다.

예시 — 닭 캐릭터:
```
몸통 = Box(0.4, 0.35, 0.35) 흰색
머리 = Box(0.25, 0.25, 0.25) 흰색
볏   = Box(0.08, 0.12, 0.08) 빨강
부리 = Box(0.08, 0.06, 0.1)  주황
다리 = Box(0.05, 0.15, 0.05) 주황 × 2
```

같은 방식으로 승용차, 트럭, 버스, 나무, 통나무도 전부 Box/Cylinder 조합으로 구현.
`flatShading: true` 옵션 하나로 면이 각지게 렌더링되어 복셀/로우폴리 스타일이 완성됨.

**길건너 친구들처럼 도형 기반의 단순한 비주얼 스타일 게임은 이 방식으로 충분히 빠르게 만들 수 있다.**

### 2.2 519KB HTML 파일 하나로 게임 전체가 동작

`vite-plugin-singlefile`로 TypeScript 13개 파일을 번들링해서 HTML 안에 인라인.
3D 모델도 코드로 만들었기 때문에 별도 파일이 없음.
결과: **index.html 하나를 다운받아서 브라우저에서 열면 바로 게임이 실행됨.**
서버도 설치도 필요 없음.

## 3. 게임 디자인 관찰

### 3.1 원작 레퍼런스와 비교

AI가 만든 결과물을 그냥 보면 "잘 만들어진 것 같은데?" 싶지만, 원작 영상/스크린샷과 나란히 놓으면 차이가 확연함.
**실제 원작 레퍼런스를 보여주고 비교하면서 수정을 진행**

실제로 수정한 것들:
| 문제 | 수정 전 | 수정 후 |
|------|--------|--------|
| 카메라 거리 | 너무 멀어서 캐릭터가 작음 | 원작 수준으로 가까이 |
| 화면비 | 16:9 가로 | 9:16 세로 (모바일 원작과 동일) |
| 밝기 | 어두운 톤 | 밝고 선명한 색감 |
| 바닥 크기 | 맵 밖이 보임 | 화면을 꽉 채움 |
| 캐릭터 방향 | 이동 반대 방향을 바라봄 | 이동 방향으로 수정 |
| 나무 충돌 | 나무를 통과함 | 나무가 장애물로 작동 |

### 3.2 게임 필(game feel)은 처음부터 자연스러운 값을 쉽게 찾진 못함

- 홉 애니메이션 높이/속도
- 카메라 따라가기 부드러움
- 차량 간격과 속도 밸런스

이런 미세한 수치는 직접 플레이해보면서 조정함
